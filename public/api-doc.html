Marketplace API Documentation üìö

Ushbu hujjat sizning marketplace backend API'ngizning barcha mavjud yo'nalishlari, ularning maqsadlari, qabul qiladigan so'rovlari va qaytaradigan javoblari haqida ma'lumot beradi.

Asosiy URL (Base URL) üåê

Sizning API'ngizning barcha yo'nalishlari quyidagi asosiy URL manzilidan boshlanadi:

http://localhost:5000/api
Eslatma: Production (ishlab chiqarish) muhitida localhost:5000 o'rniga sizning domen nomingiz ishlatiladi (masalan, https://yourdomain.com/api).

Autentifikatsiya (Authentication) üîê

Sizning API'ngizning aksariyat yo'nalishlari JWT (JSON Web Token) asosidagi autentifikatsiyani talab qiladi. Kirishda (/api/auth/login yoki /api/auth/register) olingan token har bir himoyalangan so'rovning Authorization sarlavhasida Bearer Token sifatida yuborilishi kerak.

Authorization: Bearer <your_jwt_token>

1. Autentifikatsiya Yo'nalishlari (/api/auth) üîë

Metod URL Yo'nalishi Tavsif So'rov (Request Body) Javob (Response Body) Eslatmalar

Metod URL Yo'nalishi Tavsif So'rov (Request Body) Javob (Response Body) Eslatmalar

Xatolarni Boshqarish (Error Handling) üö®

Sizning API'ngizda umumiy xatolarni boshqarish mexanizmlari mavjud:

  • 404 Not Found: Agar so'ralgan yo'nalish topilmasa, API JSON formatida xato xabarini qaytaradi. Brauzer orqali kirilganda esa public/404.html sahifasi ko'rsatiladi. 
    {
    "message": "Not Found - /api/nonexistent-route",
    "error": {
        "code": 404,
        "details": "The requested API resource could not be found."
    }
}
  • 400 Bad Request: Noto'g'ri so'rov tanasi, majburiy maydonlar yo'qligi va boshqa validatsiya xatolarida.
  • 401 Unauthorized: Autentifikatsiya tokeni taqdim etilmaganida yoki noto'g'ri bo'lganida.
  • 403 Forbidden: Foydalanuvchi so'ralgan resursga kirish huquqiga ega bo'lmaganida (masalan, admin huquqi talab qilinganda).
  • 500 Internal Server Error: Serverda kutilmagan xato yuz berganda.
